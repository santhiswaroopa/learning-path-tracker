# Minimal Code Change Skill

Purpose:
Implement fixes and features with minimal disruption to the existing codebase.

Rules:
- Modify only necessary files.
- Preserve existing UI and functionality.
- Avoid unnecessary refactoring.
- Keep changes isolated and maintainable.
- Reuse existing logic whenever possible.
- Prevent breaking unrelated features.
- Maintain backward compatibility.

Focus:
- Small targeted fixes
- Incremental improvements
- Stable code modifications
- Reduced regression risk
