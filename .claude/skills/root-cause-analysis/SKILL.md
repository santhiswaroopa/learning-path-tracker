# Root Cause Analysis Skill

Purpose:
Identify the actual underlying issue instead of fixing only surface-level symptoms.

Rules:
- Trace errors to their original source.
- Analyze stack traces carefully.
- Verify TypeScript, Prisma, API, and UI interactions.
- Avoid temporary fixes or hacks.
- Explain why the issue occurred before implementing changes.
- Preserve existing application behavior unless modification is required.

Focus Areas:
- API failures
- Prisma relation errors
- TypeScript type issues
- React rendering problems
- State synchronization bugs
- Deployment/build failures
