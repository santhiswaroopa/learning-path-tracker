# Security Analysis Skill

Purpose:
Protect the application from unsafe patterns and vulnerabilities.

Rules:
- Never expose secrets or API keys.
- Never commit .env files.
- Validate all incoming request data.
- Prevent unauthorized database access.
- Sanitize user input where necessary.
- Use hashed passwords only.
- Protect API routes requiring authentication.
- Avoid dangerous filesystem or shell operations.

Security Focus:
- Authentication
- Session validation
- API protection
- Environment variables
- Input validation
- Database safety
